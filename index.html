How to Use This Page

          

            
Getting Started

            
    
              
  1. Enter your Optimizely Project ID in the launch form. You can find this in Optimizely under Settings > Snippet.
    2. 
              
  2. Optionally provide a custom Zeotap Write Key if you have one; otherwise the default key is used.
    3. 
              
  3. Click Launch. The page reloads with the Optimizely snippet injected.
    4. 
            


            
Verification Tab

            
The verification script automatically checks that your Optimizely setup is working correctly:

            
    
              
  • Snippet Loaded — confirms the Optimizely JS snippet loaded on the page.
    • 
              
  • API Available — confirms window.optimizely.get() is callable.
    • 
              
  • Visitor ID — confirms a visitor ID was assigned.
    • 
              
  • Data Object — confirms get("data") returns project configuration.
    • 
              
  • Experiments Found — checks if any experiments are configured in the project.
    • 
              
  • Active Experiments — checks if any experiments are currently running for this visitor.
    • 
              
  • Zeotap SDK — confirms the Zeotap SDK loaded.
    • 
              
  • Zeotap Integration — confirms the visitor ID was sent to Zeotap via setPageProperties.
    • 
            


            
Experiments Tab

            
Shows a detailed table of all experiments configured in your project, including:

            
    
              
  • Experiment name, ID, and parent campaign
    • 
              
  • All variations and which one this visitor is bucketed into (highlighted)
    • 
              
  • Audience targeting information
    • 
              
  • Traffic allocation (weight distribution)
    • 
            


            
Raw Data Tab

            
Displays the full JSON response from window.optimizely.get("data"). This includes audiences, events, pages, campaigns, experiments, and all project configuration.

            

              Performance note: get("data") is a performance-intensive call. Avoid calling it repeatedly in production code. Cache the result if you need multiple fields.
            


            
Sandbox Tab

            
A JavaScript editor where you can write and execute code against the live Optimizely API. Use this to:

            
    
              
  • Test custom experiment logic
    • 
              
  • Query specific data fields
    • 
              
  • Prototype changes before implementing them
    • 
              
  • Debug audience targeting or variation assignment
    • 
            

            
Your code is wrapped in a function. Use return to output a result. The window.optimizely object and all page APIs are available.


            
Zeotap Tab

            
Interact with the Zeotap SDK directly:

            
    
              
  • Fire Event — calls zeotap.setEventProperties(name, props)
    • 
              
  • User Identities — calls zeotap.setUserIdentities() or zeotap.unsetUserIdentities()
    • 
              
  • Page Properties — calls zeotap.setPageProperties(props)
    • 
            

            

              Automatic integration: On page load, the Optimizely visitor ID is automatically sent to Zeotap as a page property (optimizely_visitor_id) using window.optimizely.get("visitor_id").
            


            
URL Parameters

            
    
              
  • ?project_id=12345678 — (required) Your Optimizely project ID
    • 
              
  • &write_key=your-key — (optional) Custom Zeotap write key
    • 
            


            
Optimizely API Reference

            
Key API calls available via window.optimizely:

            
    
              
  • get("data") — Static project config (experiments, campaigns, audiences, events, pages)
    • 
              
  • get("state") — Dynamic visitor state (active experiments, variation assignments)
    • 
              
  • get("visitor_id") — Current visitor's unique identifier
    • 
              
  • get("visitor") — Visitor profile and attributes
